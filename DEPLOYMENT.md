# Руководство по развертыванию QA Assistant

## Локальный запуск

### Простой способ (без сервера)
1. Откройте файл [index.html](./index.html) или [deepseek.html](./deepseek.html) в вашем веб-браузере
2. Приложение готово к использованию

> **Примечание**: При использовании этого способа могут возникнуть проблемы с CORS при запросах к API ИИ.

### С локальным сервером
Для корректной работы с API ИИ рекомендуется запустить приложение через локальный веб-сервер:

#### Использование Python
```bash
# Если у вас установлен Python
python -m http.server 8000
```

Затем откройте в браузере: http://localhost:8000

#### Использование Node.js
```bash
# Установка локального сервера (если не установлен)
npm install -g http-server

# Запуск сервера из папки проекта
http-server
```

#### Использование Live Server в VS Code
1. Установите расширение "Live Server" в VS Code
2. Щелкните правой кнопкой мыши по файлу [index.html](./index.html) или [deepseek.html](./deepseek.html)
3. Выберите "Open with Live Server"

## Развертывание на хостинге

### GitHub Pages
1. Закоммитьте все файлы в репозиторий GitHub
2. Перейдите в настройки репозитория
3. В разделе "Pages" выберите ветку, из которой будет осуществляться публикация
4. Ваш сайт будет доступен по адресу: `https://ваш_юзернейм.github.io/репозиторий`

### Netlify
1. Зарегистрируйтесь на [netlify.com](https://www.netlify.com/)
2. Нажмите "Add new site"
3. Выберите "Deploy with drag and drop"
4. Перетащите все файлы проекта
5. Нажмите "Deploy Site"

### Vercel
1. Зарегистрируйтесь на [vercel.com](https://vercel.com/)
2. Нажмите "Add New Project"
3. Выберите "Import Project"
4. Укажите URL вашего GitHub-репозитория
5. Нажмите "Deploy"

## Доступные версии приложения

Проект включает две версии с разными ИИ-провайдерами:

1. **OpenAI версия** ([index.html](./index.html)) - использует OpenAI GPT-3.5-turbo
2. **DeepSeek версия** ([deepseek.html](./deepseek.html)) - использует DeepSeek API через Google Apps Script

### Настройка OpenAI версии
1. Зайдите на [index.html](./index.html)
2. Введите ваш API-ключ OpenAI и нажмите "Сохранить"
3. Используйте формы для генерации тест-кейсов и баг-репортов

### Настройка DeepSeek версии
1. Зайдите на [deepseek.html](./deepseek.html)
2. Следуйте инструкциям в [DEEPSEEK_INTEGRATION.md](./DEEPSEEK_INTEGRATION.md) для настройки Google Apps Script
3. Введите ID вашего скрипта GAS и нажмите "Сохранить"
4. Используйте формы для генерации тест-кейсов и баг-репортов

## Настройка API ключей

### OpenAI
1. Зарегистрируйтесь на [platform.openai.com](https://platform.openai.com/)
2. Перейдите в раздел API-ключи (https://platform.openai.com/api-keys)
3. Создайте новый API-ключ
4. Вставьте его в OpenAI версию приложения

### DeepSeek
1. Зарегистрируйтесь на [DeepSeek](https://www.deepseek.com/)
2. Перейдите в раздел API
3. Создайте новый API-ключ
4. Следуйте инструкциям в [DEEPSEEK_INTEGRATION.md](./DEEPSEEK_INTEGRATION.md) для настройки Google Apps Script

## Возможные проблемы и их решения

### CORS ошибки
Если вы получаете ошибки CORS при запросах к API, убедитесь, что:
- Вы запускаете приложение через веб-сервер, а не как локальный файл
- Используете HTTPS (если хостинг поддерживает)
- Ваш API-ключ действителен и имеет необходимые разрешения

### Ошибки с API
- Проверьте, что ваш API-ключ введен правильно
- Убедитесь, что у вас есть достаточный баланс/квота на аккаунте
- Проверьте, что модель, которую вы используете, доступна в вашем регионе

### Проблемы с Google Apps Script (DeepSeek версия)
- Убедитесь, что скрипт опубликован как веб-приложение
- Проверьте, что в настройках доступа указано "Для всех"
- Убедитесь, что API-ключ DeepSeek введен в свойствах скрипта

## Безопасность

- **НИКОГДА** не делитесь вашим API-ключом публично
- Не коммитьте файлы с API-ключами в репозитории
- Регулярно меняйте свои API-ключи
- Используйте ограничения по IP, если это поддерживается вашим провайдером
- Для лучшей безопасности используйте DeepSeek версию, которая хранит API-ключ на стороне сервера

## Обновление приложения

Для обновления приложения до новой версии:
1. Скачайте новые файлы
2. Замените файлы на вашем хостинге
3. Обновите страницу в браузере

> **Важно**: При замене файлов ваши локально сохраненные API-ключи останутся в браузере, если вы используете ту же доменную зону.
