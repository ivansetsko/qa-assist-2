# Часто задаваемые вопросы (FAQ) о QA Assistant

## Общие вопросы

### Что такое QA Assistant?
QA Assistant - это веб-приложение, которое помогает тестировщикам создавать тест-кейсы и баг-репорты с помощью искусственного интеллекта. Приложение предоставляет два варианта: один с использованием OpenAI API, другой с использованием DeepSeek API через Google Apps Script.

### Какие функции предоставляет QA Assistant?
Приложение предоставляет две основные функции:
1. Генерация тест-кейсов по описанию функционала
2. Создание баг-репортов по описанию проблемы

### Какие ИИ-провайдеры поддерживаются?
На данный момент поддерживаются:
- OpenAI (GPT-3.5-turbo)
- DeepSeek (через Google Apps Script для безопасности)

## Вопросы по использованию

### Как начать использовать приложение?
1. Откройте соответствующую страницу ([index.html](./index.html) для OpenAI или [deepseek.html](./deepseek.html) для DeepSeek)
2. Введите ваш API-ключ (или ID скрипта для DeepSeek версии)
3. Начните использовать формы для генерации тест-кейсов или баг-репортов

### Где получить API-ключ?
- Для OpenAI: Зарегистрируйтесь на [platform.openai.com](https://platform.openai.com/) и получите ключ в разделе API-ключи
- Для DeepSeek: Зарегистрируйтесь на [DeepSeek](https://www.deepseek.com/) и получите ключ в разделе API

### Как настроить Google Apps Script для DeepSeek?
Следуйте инструкциям в файле [DEEPSEEK_INTEGRATION.md](./DEEPSEEK_INTEGRATION.md). Процесс включает создание скрипта в Google Apps Script, настройку API-ключа и публикацию веб-приложения.

## Вопросы безопасности

### Насколько безопасно использовать мой API-ключ?
- В OpenAI версии API-ключ хранится в локальном хранилище браузера, что безопасно при личном использовании
- В DeepSeek версии API-ключ хранится в Google Apps Script, что обеспечивает лучшую безопасность, так как ключ не передается клиенту

### Можно ли использовать приложение в команде?
Да, вы можете использовать приложение в команде. Для OpenAI версии каждый участник команды должен использовать свой API-ключ. Для DeepSeek версии вся команда может использовать один и тот же ID скрипта, при этом API-ключ остается защищенным.

## Технические вопросы

### Какие браузеры поддерживаются?
Приложение работает во всех современных браузерах (Chrome, Firefox, Safari, Edge).

### Возникает ошибка CORS при использовании API
Это может происходить при открытии HTML-файла напрямую в браузере. Запустите приложение через веб-сервер (локальный или удаленный).

### Почему нужно использовать Google Apps Script для DeepSeek?
Google Apps Script действует как прокси-сервер между приложением и DeepSeek API, обеспечивая безопасность API-ключа, который остается на серверной стороне.

## Проблемы и решения

### Приложение не отвечает при генерации тест-кейсов/баг-репортов
Проверьте:
- Правильность введенного API-ключа или ID скрипта
- Наличие подключения к интернету
- Доступность API (иногда API могут быть временно недоступны)

### Получаю ошибку "Invalid API key"
- Проверьте, правильно ли введен API-ключ
- Убедитесь, что API-ключ активен и не был отозван
- Для DeepSeek версии убедитесь, что правильно настроили Google Apps Script

### Генерируемые тест-кейсы/баг-репорты не соответствуют ожиданиям
AI модели могут генерировать разные результаты при разных запросах. Попробуйте:
- Уточнить или детализировать ваш запрос
- Повторно отправить запрос
- Экспериментировать с формулировками
